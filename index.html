"Principles"

  • Make the application easy to distribute: use the browser
  • Keep in mind that the intended users and audience are the (mostly) the general public: keep things as simple and as clear as possible
  • Open source: you never know from where your collaborators will come so be open
  • Try to make life easy for the developer(s): use simple tools (game engines are nice but there are learning curves)

Software

All of the applications described here use three.js, "a cross-browser JavaScript library and application programming interface used to create and display animated 3D computer graphics in a web browser using WebGL"* and several use A-Frame, "a web framework for building 3D/AR/VR experiences".

All you really need is JavaScript, HTML, and CSS.

Caveat emptor

There is a long to-do list associated to each of these applications so there are many possible features to add (and bugs to fix). Each of them I would consider at least "minimum-viable-products" (some more than that as e.g. iSpy WebGL has been around since 2014).

Also, "your mileage may vary" in terms of browser support for some features, especially for some combinations of OS and browsers.

CMS Panoramic

Idea: With A-Frame use the 360 deg panoramic images available on CDS (like these) to make an application to explore CMS and the LHC.

Usage: As a stand-alone application in the desktop or even better in a tablet or mobile as (in-principle) the view changes as you move the device. There is even a possibility for stereo VR viewing with the right equipment. You can also embed it in a page.

CMS Detector

Idea: Use the detailed 3D model of CMS made using SketchUp and three.js to make an interactive 3D application (with annotations).

Usage: A handy interactive version of the well-known image of CMS.

iSpy WebGL

Idea: Use three.js to make an event display that runs completely client-side in the browser.

Usage: CMS masterclasses, public event displays, animations, and 3D files.

Interactive event display

Idea: export the 3D scene from iSpy WebGL for an event and make it available interactively in the browser.

Usage: currently used in physics briefings (like this one) but one can imagine other uses.

N.B. Unless the real events you are showing/using have been released as part of a CMS open data release you will need approval from the collaboration at the appropriate level before you can release anything.

CMS AR

Idea: use A-Frame to make an AR application in the (mobile) browser using 3D models from CMS.

Usage: see CMS from a different perspective as a model right in front of you.
